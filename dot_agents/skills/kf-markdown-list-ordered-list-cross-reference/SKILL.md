# Markdown数字リストの相互参照

このスキルは、Markdownで数字リストを含む文章を「番号管理の手間が少なく、項目間の参照が壊れにくい」形で作成・修正するときに使います。

## このスキルを使う場面

次のような依頼で使ってください。

- Markdownのガイド、メモ、解説、手順、ランブックなど、数字リストを含む文章を書いている・直している
- 項目番号は必要だが、途中追加や並び替えのたびに手で採番し直したくない
- 後続の項目から前の項目を参照したい
- 数字リストを含むMarkdownのテンプレートがほしい
- Markdown、順序付きリスト、番号付きリスト、項目番号、アンカー、内部リンク、相互参照といった話題が出ている

## 基本ルール

### 1. 順序付きリストはソース上ですべて `1.` にする

Markdownソースでは、順序付きリストの各項目をすべて `1.` で書きます。

この書き方にする理由:

- Markdownレンダラーが表示上の連番を自動で振ってくれる
- 項目の追加や並び替えをしても、手動で番号を直す必要がない
- 差分が小さくなり、レビューしやすい

例:

```markdown
1. 端末を起動する
1. 設定ファイルを開く
1. サービスを再起動する
```

### 2. 項目の参照は表示番号ではなくアンカーで行う

後続の項目から前の項目を参照する場合は、「3番目の項目」「手順3」のように表示番号で書かず、参照先の項目冒頭にHTMLアンカーを置き、そのアンカーへリンクしてください。

書き方:

```markdown
1. <span id="item-install-db"></span>データベースをインストールする
1. [データベースのインストール項目](#item-install-db)で設定したパスワードを入力する
```

この運用にする理由:

- 項目の順番が変わっても参照が壊れにくい
- GitHub などのMarkdownプレビューで参照先へ移動しやすい
- リンク文言が「どの項目か」を表すので読みやすい

### 3. アンカー名は安定した分かりやすい名前にする

アンカー命名規則:

- 内容に合った接頭辞を付ける（例: `item-`, `section-`, `step-`）
- 内容が推測しやすい短い英単語を使う
- 単語はハイフンでつなぐ (`kebab-case`)
- 同一ドキュメント内で重複させない

良い例:

- `item-prepare`
- `section-prerequisites`
- `step-restart-service`

避ける例:

- `item-1`
- `item-before-this`
- `item-temp`

## 適用手順

ドキュメントを新規作成・修正するときは、次の順で考えてください。

1. 順序付きリストを、Markdownソース上ですべて `1.` にそろえる。
1. 後続の文脈から参照される項目、または今後参照されそうな項目を見つける。
1. その項目の本文直前に `<span id="..."></span>` を入れる。
1. 相互参照を `[準備作業](#item-prepare)` のような内部リンクへ書き換える。
1. リンク文言は表示番号ではなく、項目の内容や役割が分かる表現にする。
1. アンカー名の重複や、存在しないリンク先がないか確認する。

## 執筆テンプレート

新しい文書で数字リストを書くときは、まず次の形をベースにしてください。

```markdown
1. <span id="item-prepare"></span>準備作業を行う
1. 次のステップへ進む
1. [準備作業](#item-prepare)の結果を確認し、完了ボタンを押す
```

## 既存文書を直すときの指針

すでに `1.` `2.` `3.` のように番号が明示されている文書では:

- ソース上の番号はすべて `1.` に統一する
- 構成変更の依頼がない限り、利用者に見える項目の順番は変えない
- 「2番目の項目を参照」「手順2を参照」のような壊れやすい記述は、説明的なアンカーリンクへ置き換える

途中に新しい項目を挿入する場合は:

- 追加する項目も `1.` で書く
- 前後のソース番号は振り直さない
- 必要なときだけアンカー名やリンク文言を見直す

## 出力時の方針

Markdownを新規作成・書き直しするときは:

- ユーザーが解説を求めていない限り、完成したMarkdown本文をそのまま返す
- 特に指定がなければ、既定の出力言語は日本語にする
- 元の文書のトーンや文体は可能な限り維持する
- アンカーは必要な項目、または明らかに将来参照されそうな項目に絞る
- HTMLは `<span id="..."></span>` によるアンカー用途以外で増やさない

## 最終チェック

仕上げる前に次を確認してください。

- すべての順序付きリスト項目がソース上で `1.` になっている
- 参照される項目に一意なアンカーが付いている
- 内部リンクが対応する `#...` を向いている
- リンク文言が表示番号ではなく項目内容を説明している
- 項目の追加や並び替えがあっても、手作業の採番修正なしで維持できる
