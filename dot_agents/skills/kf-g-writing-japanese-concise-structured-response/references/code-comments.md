# コードコメントのルール

## 基本ルール

- 敬語を使わない
- 長文説明を避ける
- 目的、前提、注意点を短く書く
- 自明な処理説明を避ける

## 良い例

- `// 年次集計の起点`
- `// 取引所ごとの差分吸収`
- `// 未確定データは集計対象外`

## 避ける例

- `// ここでは年次集計の起点を設定しています。`
- `// この処理では取引所ごとの差分を吸収するための対応を行います。`

## 判断基準

- 読み手が一目で意図を追えない箇所にだけコメントを足す
- コードをなぞるだけの説明は書かない
- 抽象語より、対象領域が分かる語を優先する
- 長い補足が必要なら、コメントではなく設計や命名の見直しも検討する

## 最終チェック

- コメントが本当に必要な箇所だけに付いているか
- 一文が短く、目的がすぐ伝わるか
- コードの自明な内容をそのまま言い換えていないか
