# 順序付きリストと相互参照

## 基本ルール

1. 順序付きリストはソース上ですべて `1.` にする
1. 項目の参照は表示番号ではなくアンカーで行う
1. アンカー名は内容が分かる安定した `kebab-case` にする

## `1.` にそろえる理由

- Markdown レンダラーが表示上の連番を自動で振ってくれる
- 項目の追加や並び替えをしても、手動で番号を直す必要がない
- 差分が小さくなり、レビューしやすい

例:

```markdown
1. 端末を起動する
1. 設定ファイルを開く
1. サービスを再起動する
```

## 項目参照の書き方

後続の項目から前の項目を参照する場合は、「3番目の項目」「手順3」のように表示番号で書かず、参照先の項目冒頭に HTML アンカーを置き、そのアンカーへリンクする。

```markdown
1. <span id="item-install-db"></span>データベースをインストールする
1. [データベースのインストール項目](#item-install-db)で設定したパスワードを入力する
```

この運用にする理由:

- 項目の順番が変わっても参照が壊れにくい
- GitHub などの Markdown プレビューで参照先へ移動しやすい
- リンク文言が「どの項目か」を表すので読みやすい

## アンカー名の付け方

- 内容に合った接頭辞を付ける
- 内容が推測しやすい短い英単語を使う
- 単語はハイフンでつなぐ
- 同一ドキュメント内で重複させない

良い例:

- `item-prepare`
- `section-prerequisites`
- `step-restart-service`

避ける例:

- `item-1`
- `item-before-this`
- `item-temp`

## 適用手順

1. 順序付きリストを、Markdown ソース上ですべて `1.` にそろえる
1. 後続の文脈から参照される項目、または今後参照されそうな項目を見つける
1. その項目の本文直前に `<span id="..."></span>` を入れる
1. 相互参照を `[準備作業](#item-prepare)` のような内部リンクへ書き換える
1. リンク文言は表示番号ではなく、項目の内容や役割が分かる表現にする
1. アンカー名の重複や、存在しないリンク先がないか確認する

## テンプレート

```markdown
1. <span id="item-prepare"></span>準備作業を行う
1. 次のステップへ進む
1. [準備作業](#item-prepare)の結果を確認し、完了ボタンを押す
```

## 既存文書を直すときの指針

- すでに `1.` `2.` `3.` のように番号が明示されている文書では、ソース上の番号をすべて `1.` に統一する
- 構成変更の依頼がない限り、利用者に見える項目の順番は変えない
- 「2番目の項目を参照」「手順2を参照」のような壊れやすい記述は、説明的なアンカーリンクへ置き換える
- 途中に新しい項目を挿入する場合も、追加する項目は `1.` で書く
- HTML は `<span id="..."></span>` によるアンカー用途以外で増やさない

## 最終チェック

- すべての順序付きリスト項目がソース上で `1.` になっているか
- 参照される項目に一意なアンカーが付いているか
- 内部リンクが対応する `#...` を向いているか
- リンク文言が表示番号ではなく項目内容を説明しているか
