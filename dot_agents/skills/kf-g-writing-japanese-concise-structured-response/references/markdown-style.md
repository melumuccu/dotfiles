# Markdown 本文のルール

## 基本ルール

- 敬語を使わない
- 体言止めを多めに使う
- 一文を短く保つ
- 箇条書きと数字リストを優先
- 余分な接続詞を減らす
- 抽象語の連打を避ける

## Markdown のルール

- 見出しを使って内容を分割
- 見出しに項番を付けない
- `# 1. xxx` のような書き方をしない
- 順序が必要な内容は数字リストを使う
- 列挙だけで足りる箇所は短い箇条書きにする
- 1 項目 1 論点を基本にする

## 出力方針

- Markdown を新規作成、書き直しするときは、完成した本文を優先して返す
- ユーザーが解説を求めていない限り、文書外の説明を増やしすぎない
- 元の文書のトーンや文体は可能な限り維持する
- 項目化できる内容は本文でだらだら書かない

## 編集時の注意

- 依頼範囲外の文章まで全面的に書き換えない
- 既存情報の意味を変えない
- 表記整理より、伝達効率の改善を優先する
- 既存フォーマットに明確な制約がある場合は、その制約を優先する

## 最終チェック

- 一文が長くなりすぎていないか
- 見出しに項番を付けていないか
- 順序表現を数字リストで書いているか
- 箇条書きや数字リストが過不足なく使われているか
