# メッセージ形式

## 基本テンプレート

コミットメッセージは常に次の複数行形式で作る。

```text
<type_>: <subject (日本語)>

- 概要:
  - <何を変えたか>
  - <必要なら追加項目>
- Why:
  - <なぜこの対応を行ったか>
  - <必要なら背景や制約>
```

## 構成ルール

- 1 行目は `<type_>: <subject>` 形式にする。
- 2 行目は必ず空行にする。
- 3 行目以降に本文を書く。
- 本文は箇条書きで書く。
- 少なくとも `- 概要` と `- Why:` を含める。
- `概要` と `Why` が 1 行で収まるなら、`- 概要: ...` `- Why: ...` 形式でもよい。
- 1 文が長いときや論点が複数あるときは、見出しの下へ箇条書きを増やす。

## 5文字プレフィックス

- `type_` は 5 文字固定のプレフィックス。
- 使えるのは `feat_` `fix__` `docs_` `style` `refa_` `perf_` `test_` `chore` の 8 種類だけ。
- 一覧にない prefix を作らない。

## 本文テンプレート

迷ったら次をそのまま使う。

```text
<type_>: <subject (日本語)>

- 概要:
  - 何を変えたか
  - 必要なら次の要点
- Why:
  - なぜこの対応を行ったか
  - 必要なら背景や制約
```

## 短い例

```text
fix__: ログイン失敗時のエラーメッセージ表示を修正

- 概要: 認証失敗時にエラートーストを表示
- Why: 失敗理由が分からず再試行しづらい状態の解消
```
